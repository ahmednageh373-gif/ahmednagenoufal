# تصميم هيكل المشروع الموحد (Unified Project Structure)

لضمان التنظيم الفعال والتكامل السلس بين الأدوات والمكتبات، يجب اعتماد هيكل موحد للمشروع. هذا الهيكل يضمن سهولة الوصول إلى البيانات، فصل المدخلات عن المخرجات، وتطبيق معايير ثابتة عبر جميع المشاريع.

## الهيكل المقترح للمشروع

سيتم تنظيم كل مشروع (مثل مشروع فيلا سكنية) ضمن مجلد رئيسي، ويحتوي هذا المجلد على المجلدات الفرعية التالية:

```
/Project_Name (اسم المشروع - مثال: /Villa_AlYasmeen)
├── 01_Input_Data (البيانات المدخلة)
│   ├── 01_CAD (ملفات أوتوكاد)
│   │   ├── Drawings (الرسومات الهندسية - DWG/DXF)
│   │   └── LISP (ملفات الليسب المخصصة)
│   ├── 02_Schedule (الجدول الزمني)
│   │   ├── P6_Export (تصديرات بريمافيرا - XER/XML)
│   │   └── MS_Project (ملفات MS Project - MPP)
│   ├── 03_Cost (بيانات التكلفة)
│   │   └── BOQ_Initial (جداول الكميات الأولية - Excel)
│   └── 04_Site_Data (بيانات الموقع)
│       └── Daily_Reports (التقارير اليومية - Excel/CSV)
├── 02_Processing (المعالجة والتحليل)
│   ├── Scripts (وحدات Python البرمجية)
│   │   ├── quantity_analysis.py
│   │   ├── schedule_analysis.py
│   │   └── risk_assessment.py
│   └── Temp_Data (بيانات مؤقتة للمعالجة - CSV/JSON)
├── 03_Output_Data (البيانات المخرجة)
│   ├── 01_Quantities (الكميات النهائية)
│   │   ├── Final_BOQ (جداول الكميات النهائية - Excel)
│   │   └── QS_Calculations (حسابات الكميات التفصيلية - PDF/Word)
│   ├── 02_Reports (التقارير)
│   │   ├── Weekly_Progress (تقارير التقدم الأسبوعية - Word/PDF)
│   │   └── Smart_Reports (التقارير الذكية - Word/PDF)
│   ├── 03_Visuals (الرسوم البيانية)
│   │   └── S_Curves (منحنيات S-Curve - PNG/PDF)
│   └── 04_WBS (هيكل تجزئة العمل)
│       └── WBS_Structure (هيكل WBS المعتمد - Excel/XML)
└── README.md (ملف الوصف)

## مبادئ التنظيم

| الرقم التسلسلي | المبدأ | المواصفات | الهدف |
| :---: | :--- | :--- | :--- |
| 1 | **الفصل الواضح** | فصل البيانات المدخلة (Input) عن البيانات المخرجة (Output) عن وحدات المعالجة (Processing). | منع التعديل العرضي على البيانات الأصلية وضمان تتبع سير العمل. |
| 2 | **التسمية الموحدة** | استخدام أسماء مجلدات واضحة باللغة العربية والإنجليزية (للتوافق البرمجي) مع ترقيم تسلسلي. | سهولة التعرف على محتوى المجلدات وتسهيل كتابة مسارات الملفات في الأكواد. |
| 3 | **المركزية** | جميع الملفات المتعلقة بمشروع واحد توجد داخل مجلده الرئيسي. | ضمان استقلالية كل مشروع وتسهيل عملية الأرشفة والنقل. |
| 4 | **التتبع الآلي** | استخدام المجلد `Temp_Data` لتخزين المخرجات الوسيطة التي تنتجها وحدات Python قبل توليد التقرير النهائي. | تتبع خطوات المعالجة والتحليل والتحقق من صحة البيانات في كل مرحلة. |

هذا الهيكل يضمن أن كل مكتبة برمجية (مثل Pandas) تعرف بالضبط من أين تستمد بياناتها (مثل `01_Input_Data/03_Cost/BOQ_Initial.xlsx`) وأين تضع نتائجها (مثل `03_Output_Data/01_Quantities/Final_BOQ.xlsx`).
